HR Documents & Numbering

How to issue letters, keep the register accurate, and manage document types.

For HR team Version 1.0 Last updated July 2026

1What this module is

Every formal letter HR issues — appointment, experience, relieving, salary certificate, warning, increment, NOC, or anything ad-hoc — now gets a unique number, an issue date, an issuer name, and (optionally) a link to the PDF. Two places to work with it:

SurfaceWhat lives thereWho sees it
Employee profile → Documents tab Every document ever issued to that one employee. Issue new ones from here. Admin · HR Manager · HR Intern (read-only)
Sidebar → Documents (/hr/documents) Register of every document across the org. Search / filter / jump to any employee. Admin · HR Manager · HR Intern (read-only)

2The numbering scheme

Every doc gets a unique number in this shape:

DL - APP - 2026 - 0042
DLCompany prefix
APP3-letter type code
2026Year the doc was issued
0042Sequence for this type & year

Sequence resets per (type, year). So on 1 January 2027 the counter for every type restarts at -0001. Different types have their own sequences too — the first Experience letter of 2026 is DL-EXP-2026-0001 regardless of how many appointment letters have already been issued.

Never reused Voided numbers stay reserved. If you void DL-APP-2026-0002, the next appointment letter you issue gets DL-APP-2026-0003 — not -0002. This keeps the audit trail honest; a gap in the sequence means "this number was voided," and you can look up why in the register.

3Issuing a document

1
Open the employee's profile from Sidebar → People, then click the Documents tab.
2
Click + Issue Document (top-right of the panel).
3
Fill in the modal:
  • Document type — Appointment / Experience / etc. Pick "Other" if it's an ad-hoc letter and provide a custom label.
  • Issued on — defaults to today. Change if you're backdating.
  • Subject — one line that shows on the register row (e.g. "Joining formalities" or "1-year completion").
  • PDF link — paste the shareable Google Drive / SharePoint URL to the actual PDF.
  • Notes — anything useful for the audit trail.
4
Click Issue & allocate number. A toast pops with the allocated number, e.g. DL-APP-2026-0042.
5
Rename the PDF file to include the number (e.g. DL-APP-2026-0042 – Shubham – Appointment.pdf) and re-upload to the same Drive location. The register keeps the URL pointing to it.
Two automatic things happen 1. The employee gets an in-app notification ("Your appointment letter has been issued as DL-APP-2026-0042"). 2. If it's an appointment letter and the employee has an active onboarding checklist, the "Issue appointment letter" item auto-ticks. Same for experience / relieving letters and the offboarding checklist.

4Fixing mistakes — edit vs void

Edit (small fixes)

Use edit when the number is correct but details need tweaking: change the issued-on date, update the subject / notes, swap the PDF link. Number, type, and employee stay locked — those define the identity of the document.

Void (real mistakes)

Use void when you issued the wrong type, sent it to the wrong employee, or generated the document by mistake. Voiding requires a short reason and the row stays in the register with a red "Voided" pill. Then issue a new document — it gets the next number in the sequence.

Never delete There is no hard-delete on purpose. If the number could be freed up, an auditor couldn't tell "voided" from "never existed" — the register would lie by omission. The gap in the sequence is a feature, not a bug.

5Finding a document later

Two ways in:

  1. You know whose letter — open the employee's profile, Documents tab. All their docs are there in date order.
  2. You know the number or any keyword — go to Sidebar → Documents. The search box hits the number, the subject, the notes, and the employee name. You can also filter by type and date range.

6Managing document types

Standard types ship with the system. If you need to add, rename, hide, or reorder them, go to Config → HR Doc Types (admin + HR only).

ActionEffect
Add new typeEnter a label + optional 3-letter code. New type appears in the Issue Document dropdown immediately.
Rename labelSafe. Everyone sees the new label immediately. Past documents still resolve correctly.
Rename code (e.g. APP → APPL)Safe. Past doc numbers stay literal (DL-APP-2026-0001 keeps its "APP"). Only future issues use the new code.
ReorderUp / down arrows. Order applies to the Issue dropdown.
Hide (eye toggle)Type disappears from the Issue dropdown. Existing documents of that type still render everywhere. This is the "delete" — actual deletion is not allowed because it would orphan history.
Rule of thumb Rename freely. Hide instead of delete. If you truly want to retire a type, hide it; don't try to make the row go away.

9FAQ

Can I get the number before I've written the PDF?
Yes — the number is only allocated when you submit the modal. So the safest order is: draft the PDF locally with a placeholder name → issue in HRMS → get the number → rename the PDF using that number → upload to Drive → paste the link into the register (edit the row).
What if I issued to the wrong employee?
Void the document with reason "wrong employee" and issue a new one against the correct person. The voided row stays in the register; auditors can see what happened.
Does the system generate the PDF for me?
No. You draft the PDF wherever you already do (Word / Google Docs / a template). The HRMS holds the number and the link. Keep the PDFs in Drive / SharePoint as you already do.
What if I forget to paste the Drive link on issue?
No harm — you can edit the row later and add the link. The number is what matters; the link is a convenience.
Can I see all the letters we've issued this year?
Yes. Open Sidebar → Documents, set the type filter to "All types," set the date range to Jan 1 → today. Export via browser print if you need a snapshot.
Employee says they didn't get the notification.
Check if they have an active login (they need an app_users row linked to their employee record). The register shows the issued-by name and time regardless; the notification is a courtesy.
